## 2026-04-06 - Enable the real macOS Keychain backend for the profile secret store
- Summary: after adding a macOS compatibility smoke, the new `keyring`-backed `SystemOsSecretStore` failed to read a legacy `security` CLI item. The root cause was that `keyring` had been added without the `apple-native` feature, so the macOS build was silently using the crate's `mock` backend rather than the real Keychain implementation. Updated `Cargo.toml` to enable `apple-native` on the macOS target and recorded the lesson for future maintainers.
- Tests: `cargo test --manifest-path rust/Cargo.toml --quiet system_os_store_reads_legacy_security_cli_entries_and_writes_cli_visible_entries -- --ignored --test-threads=1`; `cargo check --manifest-path rust/Cargo.toml --quiet`; `git diff --check`
- Impact: `rust/Cargo.toml`, `rust/src/profile_secret_store.rs`, `docs/internal/ai-status.md`, `docs/internal/ai-changes.md`, `docs/internal/ai-learnings.md`
- Rollback/Risk: moderate. This is the correct fix for the macOS mock-backend issue, but it introduces the real Security Framework dependency on macOS builds; validation should keep using the ignored compatibility smoke when changing OS-secret storage again.
