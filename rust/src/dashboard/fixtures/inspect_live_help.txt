Analyze live Grafana dashboards via a temporary raw-export snapshot.

Usage: inspect-live [OPTIONS]

Options:
      --color <COLOR>
          Colorize JSON output. Use auto, always, or never. [default: auto] [possible values: auto, always, never]
      --profile <PROFILE>
          Load connection defaults from the selected repo-local profile in grafana-util.yaml.
      --url <URL>
          Grafana base URL. [default: http://localhost:3000]
      --token <API_TOKEN>
          Grafana API token. Preferred flag: --token. Falls back to GRAFANA_API_TOKEN. [aliases: --api-token]
      --basic-user <USERNAME>
          Grafana Basic auth username. Preferred flag: --basic-user. Falls back to GRAFANA_USERNAME.
      --basic-password <PASSWORD>
          Grafana Basic auth password. Preferred flag: --basic-password. Falls back to GRAFANA_PASSWORD.
      --prompt-password
          Prompt for the Grafana Basic auth password without echo instead of passing --basic-password on the command line.
      --prompt-token
          Prompt for the Grafana API token without echo instead of passing --token on the command line.
      --timeout <TIMEOUT>
          HTTP timeout in seconds. [default: 30]
      --verify-ssl
          Enable TLS certificate verification. Verification is disabled by default.
      --page-size <PAGE_SIZE>
          Dashboard search page size. [default: 500]
      --concurrency <CONCURRENCY>
          Maximum parallel dashboard fetch workers used for live inspect when supported. [default: 8]
      --org-id <ORG_ID>
          Inspect dashboards from this Grafana org ID.
      --all-orgs
          Enumerate all visible Grafana orgs and inspect dashboards across them.
      --text
          Render the live inspection analysis as plain text.
      --table
          Render the live inspection analysis as a table-oriented summary.
      --csv
          Render the live inspection analysis as CSV.
      --json
          Render the live inspection analysis as JSON.
      --yaml
          Render the live inspection analysis as YAML.
      --report [<REPORT>]
          Render a full inspection report. Defaults to flat per-query table output; use --report csv or --report json for alternate output, --report tree for dashboard-first grouped text, --report tree-table for dashboard-first grouped tables, --report dependency for dependency contracts, --report dependency-json for dependency contract JSON, --report governance for datasource governance tables, or --report governance-json for governance JSON. [possible values: table, csv, json, tree, tree-table, dependency, dependency-json, governance, governance-json]
      --output-format <OUTPUT_FORMAT>
          Alternative single-flag output selector for inspect output. Use text, table, csv, json, yaml, report-table, report-csv, report-json, report-tree, report-tree-table, report-dependency, report-dependency-json, governance, or governance-json. [possible values: text, table, csv, json, yaml, report-table, report-csv, report-json, report-tree, report-tree-table, report-dependency, report-dependency-json, governance, governance-json]
      --report-columns <REPORT_COLUMNS>
          For --report table, csv, or tree-table output, or the equivalent report-like --output-format values, limit the query report to the selected columns. Use all to expand every supported column. Supported values: org, org_id, dashboard_uid, dashboard_title, dashboard_tags, folder_path, folder_full_path, folder_level, folder_uid, parent_folder_uid, panel_id, panel_title, panel_type, panel_target_count, panel_query_count, panel_datasource_count, panel_variables, ref_id, datasource, datasource_name, datasource_uid, datasource_org, datasource_org_id, datasource_database, datasource_bucket, datasource_organization, datasource_index_pattern, datasource_type, datasource_family, query_field, target_hidden, target_disabled, query_variables, metrics, functions, measurements, buckets, query, file. JSON-style aliases like orgId, dashboardUid, dashboardTags, folderFullPath, folderLevel, folderUid, parentFolderUid, panelTargetCount, panelQueryCount, panelDatasourceCount, panelVariables, datasourceName, datasourceUid, datasourceOrg, datasourceOrgId, datasourceDatabase, datasourceBucket, datasourceOrganization, datasourceIndexPattern, datasourceType, datasourceFamily, targetHidden, targetDisabled, and queryVariables are also accepted.
      --report-filter-datasource <REPORT_FILTER_DATASOURCE>
          For --report output or report-like --output-format values, include only rows whose datasource label, uid, type, or family exactly matches this value.
      --report-filter-panel-id <REPORT_FILTER_PANEL_ID>
          For --report output or report-like --output-format values, include only rows whose panel id exactly matches this value.
      --progress
          Show a progress bar while live dashboards are fetched for inspection.
      --help-full
          Show extended help with report examples for inspect-live.
      --no-header
          Do not print headers when rendering table, csv, or tree-table inspection output, including compatible --output-format values.
      --output-file <OUTPUT_FILE>
          Write inspect output to this file.
      --also-stdout
          When --output-file is set, also print inspect output to stdout.
      --interactive
          Open the shared interactive inspect workbench over the live inspection artifacts.
  -h, --help
          Print help

Examples:

  Render governance JSON from live Grafana:
    grafana-util dashboard inspect-live --url http://localhost:3000 --token "$GRAFANA_API_TOKEN" --output-format governance-json

  Open the interactive inspect workbench over live Grafana:
    grafana-util dashboard inspect-live --url http://localhost:3000 --basic-user admin --basic-password admin --interactive
