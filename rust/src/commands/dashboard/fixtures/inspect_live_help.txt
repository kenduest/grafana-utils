Summarize live Grafana dashboards through one local analysis pass.

Usage: summary [OPTIONS]

Options:
      --color <COLOR>
          Colorize JSON output. Use auto, always, or never. [default: auto] [possible values: auto, always, never]
      --profile <PROFILE>
          Load connection defaults from the selected repo-local profile in grafana-util.yaml.
      --url <URL>
          Grafana base URL. Required unless supplied by --profile or GRAFANA_URL.
      --token <API_TOKEN>
          Grafana API token. Preferred flag: --token. Falls back to GRAFANA_API_TOKEN. [aliases: --api-token]
      --basic-user <USERNAME>
          Grafana Basic auth username. Preferred flag: --basic-user. Falls back to GRAFANA_USERNAME.
      --basic-password <PASSWORD>
          Grafana Basic auth password. Preferred flag: --basic-password. Falls back to GRAFANA_PASSWORD.
      --prompt-password
          Prompt for the Grafana Basic auth password without echo instead of passing --basic-password on the command line.
      --prompt-token
          Prompt for the Grafana API token without echo instead of passing --token on the command line.
      --timeout <TIMEOUT>
          HTTP timeout in seconds. [default: 30]
      --verify-ssl
          Enable TLS certificate verification. Verification is disabled by default.
      --page-size <PAGE_SIZE>
          Dashboard search page size. [default: 500]
      --concurrency <CONCURRENCY>
          Maximum parallel dashboard fetch workers used for live summary when supported. [default: 8]
      --org-id <ORG_ID>
          Summarize dashboards from this Grafana org ID.
      --all-orgs
          Enumerate all visible Grafana orgs and summarize dashboards across them.
      --text
          Render the live summary analysis as plain text.
      --table
          Render the live summary analysis as a table-oriented summary.
      --csv
          Render the live summary analysis as CSV.
      --json
          Render the live summary analysis as JSON.
      --yaml
          Render the live summary analysis as YAML.
      --output-format <OUTPUT_FORMAT>
          Select summary output directly. Use text, table, csv, json, yaml, tree, tree-table, dependency, dependency-json, governance, governance-json, or queries-json. [possible values: text, table, csv, json, yaml, tree, tree-table, dependency, dependency-json, governance, governance-json, queries-json]
      --report-columns <REPORT_COLUMNS>
          For table, csv, or tree-table output, limit the query report to the selected columns. Use all to expand every supported column. Supported values: org, org_id, dashboard_uid, dashboard_title, dashboard_tags, folder_path, folder_full_path, folder_level, folder_uid, parent_folder_uid, panel_id, panel_title, panel_type, panel_target_count, panel_query_count, panel_datasource_count, panel_variables, ref_id, datasource, datasource_name, datasource_uid, datasource_org, datasource_org_id, datasource_database, datasource_bucket, datasource_organization, datasource_index_pattern, datasource_type, datasource_family, query_field, target_hidden, target_disabled, query_variables, metrics, functions, measurements, buckets, query, file. JSON-style aliases like orgId, dashboardUid, dashboardTags, folderFullPath, folderLevel, folderUid, parentFolderUid, panelTargetCount, panelQueryCount, panelDatasourceCount, panelVariables, datasourceName, datasourceUid, datasourceOrg, datasourceOrgId, datasourceDatabase, datasourceBucket, datasourceOrganization, datasourceIndexPattern, datasourceType, datasourceFamily, targetHidden, targetDisabled, and queryVariables are also accepted.
      --report-filter-datasource <REPORT_FILTER_DATASOURCE>
          For structured analysis output, include only rows whose datasource label, uid, type, or family exactly matches this value.
      --report-filter-panel-id <REPORT_FILTER_PANEL_ID>
          For structured analysis output, include only rows whose panel id exactly matches this value.
      --progress
          Show a progress bar while live dashboards are fetched for summary.
      --help-full
          Show extended help with report examples for summary.
      --no-header
          Do not print headers when rendering table, csv, or tree-table summary output, including compatible --output-format values.
      --output-file <OUTPUT_FILE>
          Write summary output to this file.
      --also-stdout
          When --output-file is set, also print summary output to stdout.
      --interactive
          Open the shared interactive summary workbench over the live summary artifacts.
  -h, --help
          Print help

Examples:

  Render governance JSON from live Grafana:
    grafana-util dashboard summary --url http://localhost:3000 --token "$GRAFANA_API_TOKEN" --output-format governance-json

  Open the interactive summary workbench over live Grafana:
    grafana-util dashboard summary --url http://localhost:3000 --basic-user admin --basic-password admin --interactive
